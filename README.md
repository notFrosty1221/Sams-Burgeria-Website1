# Part 2: CSS Styling and Responsive Design

## Overview

Part 2 focused on improving the visual appearance, usability, and responsiveness of the Sams Burgeria website through CSS styling and responsive web design techniques. An external stylesheet was created and linked to all webpages to ensure consistent styling throughout the website.

---

## CSS Styling

### External Stylesheet

A single external stylesheet (`style.css`) was created and linked to all HTML pages. This approach ensured consistent styling and simplified website maintenance.

### Base Styling

The following base styles were implemented:

* Consistent font family throughout the website.
* Uniform colour scheme using burger-themed colours.
* Standardised margins and padding.
* Consistent heading and paragraph styling.
* Styled navigation menu.
* Styled footer section.
* Styled images and content sections.

### CSS Reset

A basic CSS reset was implemented to minimise browser inconsistencies and provide a consistent starting point for styling across different browsers.

Example:

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

---

## Responsive Design

### Breakpoints

Media queries were implemented to support different screen sizes:

* Desktop: 1024px and above
* Tablet: 768px – 1023px
* Mobile: 767px and below

### Layout Adjustments

Responsive techniques included:

* Multi-column layouts on larger screens.
* Single-column layouts on smaller screens.
* Responsive navigation adjustments.
* Responsive images.
* Responsive font sizing and spacing.

### Responsive Images

Responsive image techniques were implemented using:

* `srcset`
* `sizes`
* `picture` elements

These techniques allow browsers to load appropriately sized images depending on the user's device and screen resolution.

### Testing

The website was tested using browser developer tools to simulate:

* Desktop devices
* Tablet devices
* Mobile devices

Screenshots of each view were captured and included as evidence within the project documentation.

# Part 3: Enhancing Functionality and SEO

## Functionality: JavaScript Enhancements

The website was enhanced using JavaScript to improve user interaction and user experience.

### Interactive Elements

* An interactive map was implemented on the Contact page using the Leaflet JavaScript library.
* CSS animations and transitions were added to improve visual engagement.
* Dynamic content techniques were explored using JavaScript and DOM manipulation.
* A gallery with lightbox functionality was implemented to allow users to view larger versions of product images.

### Dynamic Content

* JavaScript was used to dynamically display content and improve website interactivity.
* Search functionality was implemented to allow users to search and filter products.

---

## Search Engine Optimisation (SEO)

### On-Page SEO

The following SEO techniques were implemented:

* Relevant keywords were incorporated throughout the website content.
* Unique title tags and meta descriptions were added to all pages.
* Heading tags (H1, H2, H3) were used to structure content logically.
* Images were optimised using descriptive file names and alt text.
* Internal links were implemented between pages.
* Mobile-friendly responsive design was maintained across all pages.

### Off-Page SEO

The following off-page SEO strategies were considered:

* Website links can be shared on social media platforms such as Facebook, Instagram, and WhatsApp.
* Local SEO techniques were applied by including location-based information and contact details.
* Backlink opportunities were identified through project hosting platforms and social sharing.

### Additional SEO Measures

* A robots.txt file was created to guide search engine crawlers.
* A sitemap.xml file was created to help search engines understand the website structure.
* Images were optimised to improve page loading speed.
* Basic security practices were implemented through form validation and secure coding practices.

---

## Form Functionality and Validation

### Enquiry Form

The enquiry form allows users to:

* Submit product enquiries.
* Request information regarding products and services.
* Enter validated contact information.

### Contact Form

The contact form allows users to:

* Send general messages to the organisation.
* Select a message type.
* Provide contact details and a detailed message.

### Validation

Validation was implemented using:

* HTML5 validation attributes (required, email, minlength, pattern).
* JavaScript client-side validation.
* Error handling and user feedback messages.
* AJAX-style asynchronous form submission techniques.
